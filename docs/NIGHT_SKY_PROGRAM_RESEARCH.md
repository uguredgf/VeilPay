## VeilPay fit assessment

| Criterion | Assessment | Evidence or gap |
|---|---|---|
| Core privacy need | Strong fit | Salary, identity, and employer-employee graph privacy is central to the product |
| Relevant vertical | Strong fit | Compliance and payment infrastructure are explicitly relevant program areas |
| Commercial problem | Promising, unvalidated | A plausible enterprise pain point exists, but customer interviews and pilot evidence are not documented |
| Live MVP | Partial | Functional local web demo exists; real Midnight contracts, proofs, and deployment do not |
| Midnight integration | Early | Lace connection works; transaction and proof execution are simulated |
| Team requirement | Unverified blocker | Repository contains no verified two-founder team information |
| Company stage/incorporation | Unverified blocker | Legal entity and stage are not documented |
| Financial documents | Missing blocker | Cap table, cash-flow forecast, and P&L are not in the project |
| Traction | Weak | No verified users, wallets, pilots, revenue, or transaction volume |
| Execution evidence | Moderate | Working frontend/backend, persisted flows, tests, and an explicit technical roadmap |

Concept fit is high. Current eligibility/readiness is conditional and below the published bar. The application should not claim a live on-chain MVP. Acceptance chances improve materially only if the applicant can verify a two-founder team, company/financial readiness, customer evidence, and a credible path to a compiled preprod contract.

## Application positioning

Recommended vertical: `Compliance`. `TradFi` is a defensible secondary framing, but compliance best expresses why privacy is commercially necessary.

Recommended stage: choose `Idea (Building MVP)` for the current build. Choose `Pre-seed (MVP is ready)` only if the team can provide a stable public demo and is comfortable explaining that Midnight execution remains simulated. The official page's live-MVP requirement means the first answer is honest but less eligible; the second must not be used to hide the integration gap.

Do not place “Night Sky participant” or “backed by Midnight” in product branding. Until acceptance, use only “Applying to the Night Sky Accelerator” in application materials.

## Immediate submission checklist

- Verify at least two founders and identify the technical lead.
- Confirm incorporation country and legal entity status.
- Prepare cap table, 12-month cash-flow forecast, and current P&L, even if all values are zero.
- Publish a stable demo URL and a short recorded walkthrough.
- Upload a pitch deck that labels the Midnight transaction layer as simulation.
- Replace unsupported market numbers with cited sources or remove them.
- Add customer discovery evidence only when real.
